# 문제 해결 및 일반적인 문제

SyncVRC를 사용하는 동안 문제가 발생하면 버그를 보고하기 전에 이 가이드에서 가장 일반적인 문제와 해결 방법을 확인하십시오.

## 1. AI 처리 및 하드웨어 오류

**오류:** `CUDA/GPU Error: No compatible NVIDIA GPU found.` 또는 마이크 시작 시 앱 충돌.
* **원인:** 앱이 GPU를 사용하여 Faster-Whisper 모델을 처리하려고 하지만 필요한 NVIDIA 드라이버를 찾을 수 없습니다.
* **해결 방법:**
  * **NVIDIA GPU**가 있는 경우 [NVIDIA CUDA Toolkit 12.x](./tutorial/initial-setup)를 설치해야 합니다. 설치 후 PC를 재부팅하십시오.
  * **AMD, Intel 또는 전용 GPU가 없는 경우** **Settings (설정)** 탭으로 이동하여 **AI Processing Device (AI 처리 장치)**를 `CPU (Fallback)`으로 변경하십시오.

## 2. API 및 번역 오류

**오류:** `API Limit Exceeded: Too many requests.`
* **원인:** 무료 등급 API 키(특히 Google Gemini)를 사용 중이며 너무 많이, 너무 빨리 말하여 속도 제한에 도달했습니다.
* **해결 방법:** 제한이 재설정되도록 1분 동안 말하기를 멈춥니다. 영구적인 해결을 위해서는 API 계정을 종량제 요금제로 업그레이드하십시오. 또는 SyncVRC 모드를 *Auto (Continuous)*에서 *Push 단추를 누르고 녹음(Push to Record)*으로 전환하십시오.

**오류:** `API Error: Invalid API Key.`
* **원인:** API 키가 비어 있거나, 잘못 복사되었거나, 클라우드 제공업체 계정이 제한되었습니다.
* **해결 방법:** **Settings (설정)** 탭에서 API 키를 다시 확인하십시오. 키의 시작이나 끝에 여분의 공백이 없는지 확인하십시오.

**오류:** `API Request Error: 400 Bad Request` (또는 유사한 400 오류)
* **원인:** 이는 일반적으로 제공업체에서 AI 모델 이름을 변경했거나 전송된 텍스트가 클라우드 제공업체의 안전 가이드라인(예: 극단적인 욕설)을 위반한 경우에 발생합니다.
* **해결 방법:** SyncVRC는 기본적으로 안전 필터가 비활성화되어 있지만 제공업체에서 여전히 특정 단어를 강제로 차단할 수 있습니다. 다른 문장으로 말해보십시오.

## 3. VRChat OSC 연결 문제

**문제:** OSC 상태가 `🔴 Not Detected`로 표시되거나 번역이 VRChat에 나타나지 않음.
* **원인:** VRChat이 기본 포트(`9000`)에서 OSC 데이터를 보내거나 받지 않고 있습니다.
* **해결 방법:**
  1. VRChat 방사형 메뉴(Radial Menu)를 엽니다.
  2. **Options > OSC**로 이동하여 **Off**로 전환한 다음 다시 **On**으로 켜서 연결을 재설정합니다.
  3. 다른 OSC 애플리케이션(얼굴 트래커 또는 심박수 모니터 등)이 포트 `9000` 또는 `9001`을 차단하고 있지 않은지 확인합니다.

## 4. 오디오 라우팅 및 마이크 문제

**문제:** 수신 시스템이 내 목소리나 YouTube 비디오를 변환하고 있음.
* **원인:** *Speaker Loopback (스피커 루프백)* 설정에서 물리적 스피커 또는 헤드폰을 선택했습니다.
* **해결 방법:** 가상 오디오 케이블(예: VB-Cable) 또는 Windows 스테레오 믹스를 사용해야 합니다. VRChat의 오디오**만** 이 가상 케이블로 라우팅하고 SyncVRC의 수신 설정에서 해당 가상 케이블을 선택하십시오.

**문제:** 앱에 "Listening..."이라고 표시되지만 내 목소리가 번역되지 않음.
* **원인:** AI가 사용자의 목소리를 들을 수 없거나 마이크 볼륨이 너무 낮습니다.
* **해결 방법:**
  * 마이크가 하드웨어적으로 음소거되어 있지 않은지 확인하십시오.
  * Windows 소리 설정을 확인하여 마이크가 기본 입력 장치이고 입력 볼륨이 충분히 높은지 확인하십시오.
  * 고급 오디오 설정(Advanced Audio Settings)에서 **Silence Timeout (무음 시간 초과)**을 조정하십시오. 너무 높으면 AI가 처리하기 전에 너무 오래 기다릴 수 있습니다.

## 5. 음소거 동기화가 작동하지 않음
**문제:** VRChat에서 마이크를 음소거했지만 SyncVRC가 계속 번역함.
* **원인:** OSC가 활성화되지 않았거나 Mute Sync 상자를 선택하지 않았습니다.
* **해결 방법:** **Audio Setup (오디오 설정)**으로 이동하여 **Enable VRC Mute Sync (VRC 음소거 동기화 활성화)**가 선택되어 있는지 확인하십시오. 또한 왼쪽 하단의 OSC 상태가 녹색(`🟢 Detected`)인지 확인하십시오.

## 아직 도움이 필요하신가요?

여기에 나열되지 않은 문제의 경우 설정(Settings) 탭에서 **Telemetry (Crash Reports) (원격 분석 (충돌 보고서))**가 활성화되어 있는지 확인하십시오. 이를 통해 앱이 개발자에게 오류 로그를 안전하게 보낼 수 있습니다.

또한 [GitHub Issues 페이지](https://github.com/finalsiren1/SyncVRC/issues)에서 지원을 요청하거나 버그를 보고할 수 있습니다.
