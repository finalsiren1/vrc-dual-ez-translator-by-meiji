# 현재 제한 사항 및 알려진 문제

투명성을 제공하기 위해 SyncVRC의 현재 기술적 및 실질적 제한 사항을 설명하고자 합니다. 저희는 소프트웨어가 할 수 있는 것과 없는 것에 대해 솔직해지는 것이 중요하다고 믿습니다. 이러한 제약 조건을 이해하면 적절한 기대치를 설정하고 경험을 최적화하는 데 도움이 됩니다.

## 1. VRChat 채팅창 글자 수 제한 및 지연
* **제약:** VRChat은 채팅창 메시지당 **144자**의 엄격한 제한을 부과합니다.
* **SyncVRC의 처리 방식:** 번역된 문장이 144자를 초과하는 경우 SyncVRC는 텍스트를 자동으로 더 작은 덩어리로 자릅니다. VRChat의 스팸 방지 필터를 트리거하지 않기 위해 앱은 다음 덩어리를 보내기 전에 **1.8초의 지연**을 강제합니다.
* **실제 상황:** 긴 독백이나 매우 긴 문장은 아바타 머리 위에 완전히 렌더링되는 데 몇 초가 걸립니다. 최상의 실시간 경험을 위해 문장을 비교적 짧게 유지하십시오.

## 2. API 속도 제한 (무료 등급)
* **제약:** 무료 API 키(특히 Google Gemini 무료 등급)에는 엄격한 "분당 요청 수(RPM)" 할당량이 있습니다.
* **실제 상황:** **자동(연속)** 모드를 사용하고 말을 자주 하면 매우 빠르게 이 할당량에 도달할 수 있습니다. 클라우드 제공업체가 제한을 재설정할 때까지 앱은 `API Limit Exceeded` 오류를 발생시키고 번역을 일시 중지합니다.
* **해결책:** **푸시 투 토크(Push-to-Talk)** 를 사용하여 불필요한 API 호출을 최소화하거나 API 계정을 종량제 요금제로 업그레이드하십시오.

## 3. 하드웨어 및 VR 성능 병목 현상
* **제약:** SyncVRC는 프라이버시와 낮은 대기 시간을 보장하기 위해 **Faster-Whisper (Medium)** 모델을 사용하여 로컬에서만 음성을 텍스트로 변환합니다.
* **실제 상황:**
  * **GPU 모드:** 최신 NVIDIA GPU가 있는 경우 변환은 거의 즉시 이루어집니다.
  * **CPU 모드:** AMD/Intel GPU를 사용 중이거나 CUDA를 설치하지 개 않은 경우 앱은 CPU로 대체합니다. 로컬 AI 모델을 CPU에서 실행하면서 VR에서 VRChat을 실행하는 것은 리소스를 엄청나게 많이 소비합니다. 프로세서가 충분히 강력하지 않으면 VR에서 눈에 띄는 인식 지연, PC 렉 또는 프레임 드롭이 발생할 수 있습니다.

## 4. 오디오 분리 (수신 시스템)
* **제약:** SyncVRC의 수신 시스템은 가상 오디오 케이블(Virtual Audio Cable) 또는 루프백 장치를 사용하여 오디오를 캡처합니다. 해당 장치를 통해 라우팅되는 전체 오디오 스트림을 수신합니다.
* **실제 상황:** 앱은 다른 데스크톱 소리에서 VRChat 플레이어의 음성을 **분리할 수 없습니다**. 수신 시스템이 활성화된 상태에서 YouTube를 보거나, Spotify를 듣거나, Discord 알림을 받으면 AI가 해당 배경 소음도 변환하고 번역하려고 시도하여 혼란스럽거나 무의미한 로그가 발생합니다.

## 5. 컨텍스트 및 은어 번역
* **제약:** 클라우드 AI 번역 엔진은 고속을 유지하기 위해 텍스트를 거의 문장 단위로 처리합니다.
* **실제 상황:** AI는 10분 간의 대화에 대한 장기 기억을 유지하지 않습니다. 현재 VR 세션의 완전한 시각적 및 역사적 맥락이 부족하기 때문에 맥락이 깊은 VRChat 은어, 틈새 애니메이션 용어 또는 특정 캐릭터 이름을 때때로 오역할 수 있습니다.
