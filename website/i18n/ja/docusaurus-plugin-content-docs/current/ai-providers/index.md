# クラウドAIプロバイダーとAPIキー

クラウドAIセクションへようこそ。SyncVRCでは、VRゲームのパフォーマンスに影響を与えることなく、リアルタイムで高精度な翻訳を提供するために、ハイブリッドAIアプローチを利用しています。

## SyncVRCでのAIの活用方法

翻訳パイプラインは、2つの専門的なタスクに分かれています:
1. **音声認識 (ローカル):** 音声をテキストに変換するために、完全にローカルマシン上で動作する**Faster-Whisper**モデルを使用します。これにより、瞬時の音声認識が保証されます。
2. **文脈翻訳 (クラウド):** 音声がテキスト化されると、そのテキストは高度な**クラウドAI API**（Google Gemini、DeepL、OpenAIなど）に送信され、ターゲット言語に翻訳されます。

## なぜローカルの翻訳モデルを使用しないのか？

SyncVRCがFaster-Whisperと一緒にローカルで翻訳言語モデルを実行しないのはなぜか、と疑問に思うかもしれません。その答えは**システムパフォーマンス**です。

自然で文脈を考慮した翻訳が可能な高品質の大規模言語モデル（LLM）を実行するには、膨大な計算能力とビデオRAM（VRAM）が必要です。VRChatのような要求の厳しいVRゲームとFaster-Whisperモデルに加えて、重いローカルLLMを同時に実行すると、PCのボトルネックになります。これにより、大規模なフレームドロップ、ラグが発生し、VR体験がプレイ不可能になります。

テキストの翻訳ステップをクラウドAPIにオフロードすることで、SyncVRCは以下を保証します:
* **ゲームへの影響ゼロ:** PCのリソースは、VRChatをスムーズに実行するために完全に専用のままになります。
* **超高速:** 小さなテキスト文字列をクラウドサーバーに送信するのにかかる時間はわずか数ミリ秒です。
* **優れた精度:** クラウドAPIは、軽量なローカルモデルよりもスラング、文脈、文法をはるかに良く理解する、数千億のパラメーターを持つ巨大なモデルを活用します。

## APIキーの取得

SyncVRCをこれらのクラウドサービスに接続するには、選択したAIプロバイダーの個人のAPIキーを提供する必要があります。

左側のサイドバーからプロバイダーを選択するか、**「Next（次へ）」**をクリックして、**Google Gemini APIキー**の登録と取得方法を学んでください。
