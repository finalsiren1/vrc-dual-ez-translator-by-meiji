# 現在の制限事項と既知の問題

完全な透明性を提供するために、SyncVRCの現在の技術的および実用的な制限事項の概要を説明します。私たちは、ソフトウェアができることとできないことについて率直であることが重要だと考えています。これらの制約を理解することで、適切な期待値を設定し、体験を最適化することができます。

## 1. VRChatチャットボックスの文字数制限と遅延
* **制約:** VRChatは、チャットボックスのメッセージに**144文字**の厳格な制限を課しています。
* **SyncVRCの対応:** 翻訳された文が144文字を超える場合、SyncVRCはテキストを自動的に小さなチャンクに分割します。VRChatのスパム対策フィルターをトリガーしないように、アプリは次のチャンクを送信する前に**1.8秒の遅延**を強制します。
* **実際の状況:** 長い独白や極端に長い文は、アバターの頭上に完全に表示されるまでに数秒かかります。最高のリアルタイム体験を得るために、文は比較的短く保つようにしてください。

## 2. APIレート制限（無料枠）
* **制約:** 無料のAPIキー（特にGoogle Geminiの無料枠）には、厳密な「1分あたりのリクエスト数（RPM）」の割り当てがあります。
* **実際の状況:** **自動（連続）**モードを使用して頻繁に話す場合、すぐにこの制限に達する可能性が高いです。クラウドプロバイダーが制限をリセットするまで、アプリは`API Limit Exceeded`エラーをスローし、翻訳を一時停止します。
* **解決策:** 不要なAPI呼び出しを最小限に抑えるために**プッシュ・トゥ・トーク**を使用するか、APIアカウントを従量課金プランにアップグレードしてください。

## 3. ハードウェアとVRパフォーマンスのボトルネック
* **制約:** SyncVRCは、プライバシーと低遅延を確保するために、**Faster-Whisper (Medium)**モデルを使用して音声を完全にローカルで認識します。
* **実際の状況:**
  * **GPUモード:** 最新のNVIDIA GPUをお持ちの場合、認識はほぼ瞬時に行われます。
  * **CPUモード:** AMD/Intel GPUを使用している場合、またはCUDAをインストールしていない場合、アプリはCPUにフォールバックします。VRでVRChatを実行しながら、CPU上でローカルAIモデルを実行することは非常にリソースを消費します。プロセッサが十分強力でない場合、顕著な認識遅延、PCのラグ、またはVRでのフレームドロップが発生する可能性があります。

## 4. 音声の分離（受信システム）
* **制約:** SyncVRCの受信システムは、仮想オーディオケーブルまたはループバックデバイスを使用してオーディオをキャプチャします。そのデバイスを経由するすべてのオーディオストリームをリッスンします。
* **実際の状況:** アプリはVRChatのプレイヤーの声を他のデスクトップ音から**分離することはできません**。受信システムがアクティブなときにYouTubeを見たり、Spotifyを聴いたり、Discordの通知を受け取ったりすると、AIはそのバックグラウンドノイズも認識および翻訳しようとし、混沌としたまたは意味不明なログが生成されます。

## 5. コンテキストとスラングの翻訳
* **制約:** クラウドAI翻訳エンジンは、高速を維持するためにテキストをほぼ文ごとに処理します。
* **実際の状況:** AIは、10分間の会話の長期的な記憶を保持しません。現在のVRセッションの完全な視覚的および歴史的なコンテキストが不足しているため、高度にコンテキスト化されたVRChatのスラング、ニッチなアニメ用語、または特定のキャラクター名を誤訳することがあります。
