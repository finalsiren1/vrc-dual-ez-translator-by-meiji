# 当前限制与已知问题

为了提供完全的透明度，我们要概述 SyncVRC 当前在技术和实际应用上的限制。我们认为应该坦诚地说明软件能做什么和不能做什么。了解这些限制将有助于您设定正确的期望并优化您的体验。

## 1. VRChat 聊天框字符限制与延迟
* **限制:** VRChat 对每个聊天框消息施加了严格的 **144 个字符**限制。
* **SyncVRC 的处理方式:** 如果您的翻译句子超过 144 个字符，SyncVRC 会自动将文本切分成较小的块。为了避免触发 VRChat 的防垃圾信息过滤器，应用程序在发送下一个块之前会强制进行 **1.8 秒的延迟**。
* **实际情况:** 长篇大论或极长的句子需要几秒钟才能完全显示在您虚拟形象的头顶。为了获得最佳的实时体验，请保持句子相对简短。

## 2. API 速率限制（免费层级）
* **限制:** 免费 API 密钥（尤其是 Google Gemini 免费层级）具有严格的“每分钟请求数 (RPM)”配额。
* **实际情况:** 如果您使用**自动（连续）**模式并频繁说话，您很可能会非常快地达到此配额。应用程序将抛出 `API Limit Exceeded` 错误，并暂停翻译，直到云服务提供商重置您的限制。
* **解决方案:** 使用**按键说话（Push-to-Talk）**以尽量减少不必要的 API 调用，或将您的 API 账户升级为按量付费的计费计划。

## 3. 硬件与 VR 性能瓶颈
* **限制:** SyncVRC 完全在本地使用 **Faster-Whisper (Medium)** 模型来转录您的声音，以确保隐私和低延迟。
* **实际情况:**
  * **GPU 模式:** 如果您有现代的 NVIDIA GPU，转录几乎是瞬间完成的。
  * **CPU 模式:** 如果您使用的是 AMD/Intel GPU 或尚未安装 CUDA，应用程序将回退到使用您的 CPU。在运行 VR 中的 VRChat 的同时在您的 CPU 上运行本地 AI 模型是非常消耗资源的。如果您的处理器不够强大，您可能会在 VR 中体验到明显的转录延迟、电脑卡顿或掉帧。

## 4. 音频隔离（接收系统）
* **限制:** SyncVRC 的接收系统使用虚拟音频线（Virtual Audio Cable）或回环设备捕获音频。它会监听通过该设备路由的所有音频流。
* **实际情况:** 应用程序**无法**将 VRChat 玩家的声音与其他桌面声音隔离开来。如果在接收系统处于活动状态时观看 YouTube、听 Spotify 或接收 Discord 通知，AI 将尝试转录和翻译该背景噪音，从而导致混乱或无意义的日志。

## 5. 语境和俚语翻译
* **限制:** 云 AI 翻译引擎为了保持高速度，几乎是逐句处理文本的。
* **实际情况:** AI 不会保留 10 分钟对话的长期记忆。由于它缺乏您当前 VR 会话的完整视觉和历史背景，它可能偶尔会误译高度语境化的 VRChat 俚语、小众动漫术语或特定角色名称。
